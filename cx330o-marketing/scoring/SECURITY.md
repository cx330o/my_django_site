# Security Policy

Goals of the cx330o Lead Scoring Security Team
---------------------------------

*   Resolve reported security issues in a Security Advisory
*   Provide documentation on how to write secure code
*   Provide documentation on securing your cx330o Lead Scoring instance
*   Help the infrastructure team to keep the \*.cx330o Lead Scoring.org infrastructure secure

Scope of the cx330o Lead Scoring Security Team
---------------------------------

The cx330o Lead Scoring Security Team operates with a limited scope and only directly responds to issues with cx330o Lead Scoring core, officially supported plugins and resources, and the \*.cx330o Lead Scoring.org network of websites. The team does not directly handle potential vulnerabilities with third party plugins or individual cx330o Lead Scoring instances.

Which releases get security advisories?
---------------------------------------

Check the [Releases page](https://www.cx330o Lead Scoring.org/cx330o Lead Scoring-releases) to find which are the currently supported releases.

Security advisories are only made for issues affecting stable releases in the supported major version branches. That means there will be no security advisories for development releases, alphas, betas or release candidates.

Security issues are resolved in all affected versions. For example if a security issue is found in cx330o Lead Scoring 6.0.2 and it also affects all versions back to 5.2.1, then the fix will be made in the next security releases for cx330o Lead Scoring 5 and the cx330o Lead Scoring 6 release.

Once a cx330o Lead Scoring version is out of security support, it's possible to purchase [Extended Long Term Support](https://mau.tc/elts) subscriptions on a per-instance or volume license basis in blocks of one year, for up to two years. The Security Team backports any relevant fixes to the currently supported ELTS supported versions, and fixes any issues identified for those versions after security support has ended, with advisories shared via the main cx330o Lead Scoring repository. ELTS fixes are only available to current holders of Extended Long Term Support subscriptions.

How to report a potential security issue
----------------------------------------

If you discover or learn about a potential error, weakness, or threat that can compromise the security of cx330o Lead Scoring and is covered by the [Security Advisory Policy](https://www.cx330o Lead Scoring.org/cx330o Lead Scoring-security-team/cx330o Lead Scoring-security-advisory-policy), we ask you to keep it confidential and submit your concern to the cx330o Lead Scoring security team.

To make your report please submit it as a private disclosure via the relevant repository's security tab - for this repository, that's at [https://github.com/cx330o Lead Scoring/cx330o Lead Scoring/security](https://github.com/cx330o Lead Scoring/cx330o Lead Scoring/security). You can also create a private fork to provide a fix, if you're able to do so. See the documentation from GitHub on [privately reporting a security issue](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability).

Do not post it in GitHub as an issue or a Pull Request, on the forums, or discuss it in Slack.

[Read more: How to report a security issue with cx330o Lead Scoring](https://www.cx330o Lead Scoring.org/cx330o Lead Scoring-security-team/how-to-report-a-security-issue)

How are security issues resolved?
---------------------------------

The cx330o Lead Scoring Security Team are responsible for triaging incoming security issues relating to cx330o Lead Scoring core and officially supported plugins and resources, and for releasing fixes in a timely manner.

[Read more: How are security issues triaged and resolved by the cx330o Lead Scoring Security Team?](https://www.cx330o Lead Scoring.org/cx330o Lead Scoring-security-team/triaging-and-resolving-security-issues)

How are security fixes announced and released?
----------------------------------------------

The Security Team coordinates security announcements in release cycles and evaluates whether security issues are ready for release several days in advance.

The team may deem it necessary to make an out-of-sequence release, in which case at least two weeks’ notice will be provided to ensure that cx330o Lead Scoring users are made aware of a security release being made on an unscheduled basis.

[Read more: Security fix announcements and releases](https://www.cx330o Lead Scoring.org/cx330o Lead Scoring-security-team/triaging-and-resolving-security-issues)

What is a security advisory?
----------------------------

A security advisory is a public announcement managed by the cx330o Lead Scoring Security Team which informs cx330o Lead Scoring users about a reported security problem in cx330o Lead Scoring core or an officially supported plugins and resources, and the steps cx330o Lead Scoring users should take to address it. (Usually this involves updating to a new release of the code that fixes the security problem.)

[Read more: cx330o Lead Scoring Security Advisory Policy](https://www.cx330o Lead Scoring.org/cx330o Lead Scoring-security-team/cx330o Lead Scoring-security-advisory-policy)

What is the disclosure policy of the cx330o Lead Scoring Security Team?
----------------------------------------------------------

The Security Team follows a Coordinated Disclosure policy: we keep issues private until there is a fix. Public announcements are made when the threat has been addressed and a secure version is available.

When reporting a security issue, observe the same policy. **Do not** share your knowledge of security issues with others.

How do I join the cx330o Lead Scoring Security Team?
---------------------------------------

As membership in the team gives the individual access to potentially destructive information, membership is limited to people who have a proven track record in the cx330o Lead Scoring community.

Team members are expected to work at least a few hours every month. Exceptions to that can be made for short periods to accommodate other priorities, but people who can't maintain some level of involvement will be asked to reconsider their membership on the team.

[Read more: How do I join the cx330o Lead Scoring Security Team?](https://www.cx330o Lead Scoring.org/cx330o Lead Scoring-security-team/join-the-team)
